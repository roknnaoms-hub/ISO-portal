# ISO 통합 포탈 세부 작성계획(확장형)

## 1. 목적
1차로 ISO 9001, ISO 14001, ISO 42001, ISO 45001의 요구사항 4.1 ~ 10.2(3) 범위를 구축하고, 이후 ISO 27001, ISO 27701, ISO 22301, ESG auditor 영역까지 무중단으로 확장 가능한 구조를 설계한다.

- ISO 요구사항 표준
- 조직담당자 중점사항
- 인증심사원 중점사항
- 결함사례

## 2. 범위 및 작성 단위
### 2.1 표준 범위
#### 1차 구축
- ISO 9001
- ISO 14001
- ISO 42001
- ISO 45001

#### 2차 확장(향후)
- ISO 27001
- ISO 27701
- ISO 22301
- ESG auditor

### 2.2 요구사항 범위
- 4.1 ~ 10.2(3)
- 공통 장 구조(4~10장) 기준으로 조항 단위 작성
- 세부 항목은 조항, 하위 조항, 필요 시 항목(1), (2), (3)까지 분해

### 2.3 포탈 게시 단위
- 기본 단위: 표준 x 조항(예: ISO 9001 - 8.5.1)
- 각 단위당 4개 콘텐츠 블록을 동일 구조로 게시

### 2.4 확장 단위(Framework 단위)
- 표준별 상위 구분값(framework)을 도입한다.
- framework 예시: ISO9001, ISO14001, ISO42001, ISO45001, ISO27001, ISO27701, ISO22301, ESG_AUDITOR
- 신규 표준 추가 시 UI/DB 구조 변경 없이 framework 등록만으로 확장한다.

## 3. 소스 문서 및 추출 원칙
### 3.1 소스 문서
#### A. ISO 요구사항 표준용
- 표준 원문 또는 강의교재 요구항목 텍스트
- 현재 워크스페이스 기준 후보 파일:
  - ISO9001/ISO9001-2015 표준.pdf
  - ISO14001/ISO 14001-2026 Final Draft - 한글.pdf
  - ISO42001/ISO42001 요구사항 한글본 v2-241116.pdf
  - ISO45001/ISO 45001 2018 요구사항(교재).pptx
  - 필요 시 영문 원문 대조 파일(예: ISO_IEC_42001_2023(en).pdf)

#### B. 나머지 3개 콘텐츠용
- 인증심사원 안내서 v1.0 도서 텍스트
- 대상 콘텐츠:
  - 조직담당자 중점사항
  - 인증심사원 중점사항
  - 결함사례

#### C. 인증서/지정서 근거 관리용
- 파일: 한국사이버리터러시연구원 지정서_1차 사후_42001 추가.pdf
- 용도: 기관 보유 자격/지정 범위 근거 문서 관리
- 적용: 포탈의 표준별 활성화 기준 및 신뢰성 메타데이터로 활용

### 3.2 추출 원칙
- 원칙 1: 조항 번호 정합성 우선
- 원칙 2: 원문 의미 보존, 문장 최소 편집
- 원칙 3: 출처(파일명/페이지/슬라이드) 필수 기록
- 원칙 4: 중복 내용은 통합하되 표준별 맥락 차이는 유지
- 원칙 5: 직역보다 현업 이해 가능성 우선(단, 기술적 의미 불변)

### 3.3 출처 우선순위
- ISO 요구사항 표준: 표준 원문 > 강의교재
- 중점사항/결함사례: 인증심사원 안내서 v1.0 우선 단일 출처

## 4. 데이터 구조(포탈 공통 스키마)
조항별 데이터는 아래 필드를 공통으로 사용한다.

- framework: ISO9001 | ISO14001 | ISO42001 | ISO45001 | ISO27001 | ISO27701 | ISO22301 | ESG_AUDITOR
- framework_type: ISO_MSS | PRIVACY | BCM | ESG
- standard_code: 예) ISO 9001:2015, ISO/IEC 27701:2019
- version: 예) 2015, 2018, 2023, 내부판본 v1.0
- language: KO | EN
- clause_id: 예) 4.1, 8.5.1, 10.2(3)
- clause_title: 조항 제목
- clause_path: 상위-하위 조항 경로(정렬/탐색용)
- content_type: requirement | org_focus | auditor_focus | defect_case
- iso_requirement_text: 요구사항 표준 텍스트
- org_focus_points: 조직담당자 중점사항
- auditor_focus_points: 인증심사원 중점사항
- defect_cases: 결함사례
- source_standard: 표준/교재 출처 정보(파일, 페이지)
- source_guide: 안내서 출처 정보(파일, 페이지)
- source_certificate: 지정서/인증서 출처 정보(파일, 발급일, 유효기간, 적용범위)
- keywords: 검색 키워드
- related_clauses: 연계 조항
- review_status: draft | review | approved
- last_updated: 수정일
- owner: 담당자
- tags: 도메인 태그(리스크, 거버넌스, 통제, 법규, 성과 등)
- visibility: public | internal

## 5. 작성 템플릿(조항 1건 기준)
아래 템플릿으로 모든 조항을 동일하게 작성한다.

### [표준명] [조항번호] [조항제목]
- 1) ISO 요구사항 표준 원문 추출(선행)
  - 원문 추출 text:
    - (표준/교재에서 추출한 원문 또는 준원문 전체)
  - 원문 정제 text:
    - (오탈자/줄바꿈/표기만 정리한 준원문)
- 2) 조직담당자 중점사항(원문 기반 재구성)
  - (실행 관점 체크포인트)
  - 항목 수 가이드: 최소 3개, 상한 없음(10개 이상 허용)
- 3) 인증심사원 중점사항(원문 기반 재구성)
  - (심사 시 확인 포인트)
  - 항목 수 가이드: 최소 3개, 상한 없음(10개 이상 허용)
- 4) 결함사례(원문 기반 재구성)
  - 사례 [번호]: [사례명]
    - 원인:
    - 심사증거/정황:
    - 리스크/영향:
    - 시정조치/재발방지:
  - 항목 수 가이드: 최소 1개, 상한 없음(10개 이상 허용)
- 5) 출처
  - 표준/교재: 파일명, 페이지(또는 슬라이드)
  - 안내서: 파일명, 페이지
  - 지정서/인증서(해당 시): 파일명, 발급일, 유효기간, 적용범위

### 5.1 작성 규칙(항목 수 확장 대응)
- 중점사항과 결함사례는 조항 난이도에 따라 10개 이상 작성 가능하다.
- 개수 제한 대신 품질 기준을 적용한다(중복 금지, 실행 가능성, 증거 연결성).
- 결함사례는 가능한 경우 유형 태그를 부여한다(문서화 미흡, 실행 미흡, 검증 미흡, 법규 미준수 등).
- 긴 목록은 우선순위 라벨(P1/P2/P3) 또는 중요도(High/Medium/Low)를 부여한다.

### 5.2 원문 선추출 결과 파일(현재)
- 자동 1차 추출: 추출_원문_초안.md
- 자동 2차 조항탐지: 추출_원문_조항탐지_2차.md
- 활용 방법:
  - 조항별 작성 전, 2차 파일에서 해당 조항 패턴 검색 후 원문 근거를 수집한다.
  - 수집한 원문을 템플릿의 "원문 추출 text"에 먼저 기록한 뒤 중점사항/결함사례를 작성한다.

## 6. 세부 작성 절차
### 단계 1. 조항 인덱스 확정
- 4.1~10.2(3)까지 표준별 조항 목록 확정
- 표준 간 번호 체계 차이(신설/누락/표현차) 표시

### 단계 2. 소스 전처리
- PDF/PPTX 텍스트 추출 가능성 확인
- OCR 필요 페이지 식별
- 페이지 번호 기준 참조 체계 통일

### 단계 3. ISO 요구사항 표준 텍스트 추출
- 표준/교재에서 조항별 텍스트 추출
- 문장 단위로 정제(줄바꿈/각주/머리말 제거)
- 조항 번호와 제목 검증

### 단계 4. 안내서 텍스트 추출(3개 콘텐츠)
- 인증심사원 안내서 v1.0에서 조항별 관련 텍스트 추출
- 조직담당자/심사원/결함사례로 분류 태깅

### 단계 5. 조항별 초안 작성
- 조항 단위로 4개 콘텐츠 블록 작성
- 중복 표현 통합 및 누락 여부 확인

### 단계 6. 교차 검증
- 표준 원문과 의미 일치 검토
- 안내서 인용 위치 재확인
- 오탈자, 조항번호, 출처 누락 점검

### 단계 7. 품질 리뷰
- 내부 리뷰(콘텐츠 정확성)
- 심사 관점 리뷰(실무 적용성)
- 승인 상태 변경(review -> approved)

### 단계 8. 포탈 업로드
- 공통 스키마에 맞게 업로드
- 검색 키워드 및 연계 조항 연결

### 단계 9. 운영/개정 관리
- 표준 개정 및 안내서 버전 변경 시 영향도 분석
- 변경 조항 재작성 후 이력 관리

### 단계 10. 신규 표준 온보딩(확장 절차)
- Step 10-1: framework 등록(코드, 버전, 언어, 도메인)
- Step 10-2: 조항 체계 매핑(기존 4~10 구조와 차이 식별)
- Step 10-3: 소스 우선순위 정의(표준/교재/안내서/지정서)
- Step 10-4: 3개 샘플 조항 파일럿 작성 및 품질 기준 검증
- Step 10-5: 전체 조항 확장 배포

## 7. 역할 및 책임(RACI)
- 총괄 PM: 일정/품질/승인 관리
- 표준 추출 담당: ISO 요구사항 표준 텍스트 추출 및 정합성 검증
- 안내서 추출 담당: 3개 콘텐츠 텍스트 추출 및 분류
- 도메인 리뷰어(조직 관점): 조직담당자 중점사항 검증
- 도메인 리뷰어(심사 관점): 인증심사원/결함사례 검증
- 포탈 운영 담당: 데이터 업로드, 태그, 검색, 배포

## 8. 표준별 작성 중점
### ISO 9001
- 프로세스 접근, 성과지표, 고객만족, 변경관리 중심으로 중점사항 정리

### ISO 14001
- 환경측면, 준수의무, 리스크/기회, 운영관리 및 비상대응 관점 강화

### ISO 42001
- AI 시스템 거버넌스, 책임, 투명성, 위험통제 및 라이프사이클 관리 반영

### ISO 45001
- 위험성평가, 근로자 참여, 운영통제, 사고/시정조치 실효성 중심 구성

### ISO 27001(확장)
- 정보보호 리스크 처리, 통제 목적, 적용가능성 선언(SoA) 연계 중심 구성

### ISO 27701(확장)
- 개인정보 처리자/관리자 역할, PII 라이프사이클, 프라이버시 통제 연계 강화

### ISO 22301(확장)
- BIA, RTO/RPO, 비즈니스 연속성 전략 및 훈련/테스트 실효성 중심 구성

### ESG auditor(확장)
- 환경/사회/거버넌스 지표 정의, 데이터 신뢰성, 검증증적 추적성 중심 구성

## 9. 인증서(지정서) 확인 및 반영 기준
- 확인 파일: 한국사이버리터러시연구원 지정서_1차 사후_42001 추가.pdf
- 현재 확인 가능 항목: 파일 존재, 파일명, 수정시각, 파일크기
- 현재 한계: 작업 환경에서 PDF 본문 텍스트 직접 추출 불가(이미지/OCR 가능성)
- 반영 정책:
  - 인증서 원문 텍스트는 OCR 또는 수기 확인 후 `source_certificate`에 등록
  - 등록 필수 필드: 발급기관, 발급일, 유효기간, 적용 표준, 적용 범위, 증빙 파일
  - 표준 활성화 조건에 인증서 유효성 체크를 연결

## 10. 품질 기준(완료 정의)
아래 조건을 모두 충족하면 조항 1건 완료로 본다.

- 4개 콘텐츠 블록이 모두 작성됨
- 조항 번호/제목이 원문과 일치함
- 출처(파일+페이지/슬라이드)가 모두 기재됨
- 중점사항은 실행 가능한 문장으로 작성됨
- 결함사례는 원인-증거-개선 연결이 확인됨
- 리뷰 상태가 approved로 변경됨

## 11. 리스크 및 대응
- 리스크: PDF OCR 품질 저하
  - 대응: 수동 교정, 이중 검수
- 리스크: 조항 매핑 오류
  - 대응: 조항 인덱스 잠금 후 작성, 변경 이력 관리
- 리스크: 중복/상충 기술
  - 대응: 리뷰 체크리스트 기반 통합
- 리스크: 일정 지연
  - 대응: 표준별 병렬 작업, 주간 마일스톤 점검

## 12. 일정 예시(10주, 확장형)
- 1주차: 1차 표준 범위 확정, 조항 인덱스, 소스 전처리
- 2~3주차: 1차 표준 요구사항 텍스트 추출 완료
- 4~5주차: 안내서 기반 3개 콘텐츠 추출/분류 완료
- 6주차: 1차 표준 조항별 초안 작성 완료
- 7주차: 교차검증 및 품질리뷰
- 8주차: 1차 포탈 업로드, 최종 승인
- 9주차: 확장 표준(27001/27701/22301/ESG) 온보딩 설계
- 10주차: 확장 표준 파일럿 3조항씩 작성 및 게이트 리뷰

## 13. 즉시 실행 체크리스트
- 표준별 4.1~10.2(3) 조항 목록표 먼저 작성
- 추출 파일명/페이지 메타데이터 양식 확정
- 조항 3건 파일럿 작성 후 템플릿 보정
- 표준별 병렬 작성 킥오프
- framework 등록 테이블(코드/버전/언어/도메인) 생성
- 인증서 메타데이터 템플릿(발급일/유효기간/범위) 생성
